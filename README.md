Sistema de Gestión de Reservas y Servicios
Prácticas Simuladas de Programación Orientada a Objetos y Manejo de Excepciones

Este proyecto fue desarrollado como parte de las prácticas simuladas de programación, enfocándose en la aplicación de conceptos avanzados de Programación Orientada a Objetos (POO) y manejo robusto de excepciones en Python, garantizando la estabilidad, continuidad y correcto funcionamiento del sistema ante posibles errores durante su ejecución.

El sistema simula un entorno de gestión de reservas y servicios sin utilizar bases de datos, permitiendo registrar clientes, crear servicios especializados y procesar reservas, aplicando validaciones, herencia, polimorfismo, encapsulamiento y control avanzado de errores.

Objetivo del Proyecto

Desarrollar una aplicación orientada a objetos capaz de manejar operaciones válidas e inválidas de manera controlada, manteniendo la ejecución activa mediante el uso profesional de excepciones y registros de errores.

Características Principales

✔️ Implementación completa de Programación Orientada a Objetos
✔️ Manejo avanzado de excepciones personalizadas
✔️ Uso de bloques:

try / except
try / except / else
try / except / finally
Encadenamiento de excepciones

✔️ Registro automático de errores en archivos de logs
✔️ Simulación de operaciones reales sin base de datos
✔️ Validaciones robustas de datos
✔️ Aplicación de herencia, abstracción y polimorfismo
✔️ Métodos sobrecargados para cálculos dinámicos
✔️ Sistema estable incluso ante errores graves
